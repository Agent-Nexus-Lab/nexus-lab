# MaaS 活动信息抽取测评报告

测评日期：2026-05-22

## 1. 测评范围

输入目录：`experiments/agent-maas-cli/texts/`

输出文件：`experiments/agent-maas-cli/outputs/events.json`

运行命令：

```powershell
python experiments\agent-maas-cli\run_eval.py --range 1-5 --base-url https://api.modelarts-maas.com/openai/v1 --thinking disabled --max-tokens 16000 --delay-seconds 22 --retries 1
```

参数说明：

- `--max-tokens 16000`：`4.txt` 长文输出较长，低输出上限会截断 JSON。
- `--delay-seconds 22`：本账号 MaaS 返回过 `3 times per minute` 限流，本次显式避开该限制。
- `--retries 1`：`3.txt` 首次调用曾偶发空结果，重试后恢复。

## 2. JSON 校验

已用 `schema.validate_events_file` 校验 `outputs/events.json`，结果通过。

当前结构：

```json
{
  "events": []
}
```

校验结论：

- 顶层只包含 `events`。
- `events[*].event_id` 均为 UUIDv4。
- `event_id` 唯一。
- 时间字段没有纯日期字符串；仅有日期、没有具体时间的候选使用 `null`。
- 未输出 `quality_score`、`verification_status`、`is_user_visible`、`reason_text`、`display_order` 等非抽取阶段字段。

## 4. 逐样例评估

### `1.txt`

原文是复旦天协“路边天文 | 春末夏初，群星交替”，时间为 5月15日 20:00，地点为光草东北角。

输出正确抽取标题、时间、地点、组织方和证据片段。原文未明确结束时间、校区、URL，输出为 `null` 合理。

### `2.txt`

原文是复旦大学天文协会 25 周年社庆，时间为 2026.5.10 19:00-21:00，地点为叶耀珍学生活动中心。

输出正确抽取起止时间、地点和活动内容。没有根据建筑名推断校区，`campus=null` 符合约定。

### `3.txt`

原文是 FDU燕园剧社“即兴编演工作坊暨《第三方指控》剧组分享会”，时间为 2026.04.19 18:30-20:30，地点为青书馆 room8-10。

最终输出正确。问题是全量运行首轮曾返回空事件，重试后恢复，说明模型仍有偶发遵循不稳定；脚本已提供可选 `--retries`，但这不是根治。

### `4.txt`

原文是复旦大学 2026 年校园开放日长文，包含论坛、展评、招生咨询、社团嘉年华、草坪音乐节、宿舍参观、校园导览、实验室参观、餐券、抽奖等多个事项。

输出 23 条，覆盖主要活动，长文不再失败。时间、地点、江湾校区和 evidence 大体可回溯；校园导览拆成两个批次是合理的；科普实践参观没有具体时间，输出 `start_time=null`、`end_time=null` 合理。

主要问题是边界偏宽：午餐餐券、集章抽奖、寄给未来的信属于福利/互动服务，是否入库为正式活动需要后续产品或审核规则决定。

### `5.txt`

原文是返校日图书馆活动，核心活动为“点亮一本书的生命”线下专场，另有转发集赞福利和图书馆开放参观。

输出 3 条候选：主活动、集赞福利兑换、图书馆开放参观。主活动和兑换福利的时间地点正确。图书馆开放参观原文只有日期，没有具体起止时间，最终输出被归一为 `null`，符合“不得补全天时间”的约定。

## 5. 总体判断

当前结果已经适合作为抽取原型测评材料：结构符合 5.5 节，能从短文本和长文本中抽取多条候选，并保留来源文件、来源名称、证据片段和 UUIDv4 `event_id`。

仍需注意：

- MaaS 对长文本输出存在截断风险，批量测评时应按样例规模显式设置输出上限。
- MaaS 有账号侧频率限制，批量测评可能需要显式 `--delay-seconds`。
- 模型偶发空结果，`--retries` 可缓解，但未来更稳的做法是分块抽取或加入确定性后处理。
- “活动候选”和“福利/服务/配套互动”的边界还需要团队定义，当前原型先保留为候选，留给人工复核或后续规则筛选。
