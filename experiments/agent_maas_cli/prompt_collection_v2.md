# 活动采集抽取 (Collection V2) — 少字段 + 丰富 summary

从 `source_text` 抽取校园活动事实，输出精简事件（6 字段）+ 丰富的自然语言 summary 画像。

输入字段：`title`、`source_url`、`source_name`、`publish_time`、`reference_date`、`timezone`、`source_text`。

## 输出 JSON Schema

```json
{
  "events": [
    {
      "title": "活动标题（原文核心信息）",
      "summary": "活动语义画像（详见下方要求）",
      "start_time": "2026-06-28T14:00:00+08:00",
      "end_time": "2026-06-28T17:00:00+08:00",
      "location": "江湾校区计算机楼一楼大厅",
      "source_url": "https://mp.weixin.qq.com/s/xxx"
    }
  ],
  "warnings": [],
  "status": "ok"
}
```

## 字段要求

### title（必填）
- 从原文提取活动标题或核心信息
- 原文无明确标题时，用「主办方 + 活动类型」组合（如"复旦AI社AI展览"）

### summary（必填，重点）
summary 是一段 **80-200 字的自然语言活动语义画像**，不是一句话摘要。必须覆盖以下维度（缺失的维度注明"未提及"即可）：

1. **活动内容**：这是什么活动、由谁主办、主题是什么
2. **活动形式**：讲座/展览/工作坊/比赛/聚会/演出/路演/其他
3. **适合人群**：面向什么专业/年级/兴趣群体
4. **氛围与风格**：正式还是轻松、学术还是娱乐、安静还是热闹
5. **商业化程度**：公益免费/学校主办/商业推广/收费参与
6. **互动性与参与方式**：是否需要报名、是否有互动环节、参与成本（时间/金钱）
7. **可能不适合的情况**：如果活动有排他性（仅限某专业/某年级）、需要前置知识、不适合休闲场景等，在文本中自然说明

**要求**：
- 基于原文事实，不编造信息
- 缺失维度如实写"未提及具体的XX信息"
- 用自然流畅的段落，不用列表格式
- 为后续 embedding 检索和推荐解说提供足够语义

### start_time / end_time
- ISO 8601 带时区，如 `2026-06-28T14:00:00+08:00`
- 原文没有明确时间时填 `null`，不得用 00:00:00 补全
- 相对日期以 `reference_date` 为基准推算

### location
- 原文的地点描述，保留原文表述
- 可包含校区信息（如"江湾校区计算机楼一楼大厅"）
- 原文无地点时填 `null`

### source_url
- 输入提供时原样回传
- 无来源链接时填 `null`

## 失败处理

当以下情况发生时，`status` 设为对应值，`warnings` 中记录原因：

| status | 含义 |
|---|---|
| `ok` | 成功抽取 ≥1 个活动 |
| `no_activity` | 原文包含活动通知但无法提取结构化信息 |
| `not_an_event` | 原文不含任何活动信息（纯新闻/通知/广告） |
| `text_too_short` | 正文太短（<100 字），无法判断 |
| `parse_error` | 模型无法理解或解析 |

## 规则

1. 只抽取原文事实，不得编造。原文可能来自网页抓取，含导航/广告/页脚等噪声，应忽略。
2. 同一文章可抽取多个活动，每个活动一个 event 对象。
3. 缺失字段填 `null`，不要编造。
4. 时间不明确时填 `null`。相对日期以 reference_date 推算。
5. 不输出 event_id（由入库服务生成）。
6. 每个 event 的 source_url 与输入一致（同一来源）。
7. 只输出 JSON，不要输出额外文字。
